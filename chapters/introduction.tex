\section{Scientific writing advice}
Tips on \LaTeX~writing:
\begin{itemize}
\item NEVER: use $ words $ in math mode!!!!!!!!!!
\item Embed equations nicely into your text. An example is an equation, like
\begin{equation}
	x = 0,
\end{equation}
that has proper punctuation around it and is smoothly integrated into a sentence.
\item Use headlines and \verb|\begin{itemize}\item something\end{itemize}| for early stage drafts.
\item For verbose naming of variables in math mode use: \verb|\textrm{}|, instead of $ simulation tolerance = $ use something like $ \varepsilon_{\textrm{tol}} $, instead of $ xref $, use $ x\ind{ref} $, etc..
\item Tip: Use \verb|\newcommand{}| to define variables and common expressions. It is especially useful if the the notation might change later. See \verb|setup.tex| for examples
%\item Examples for some of the commands defined in \verb|setup.tex| are: $ x\ind{ref}\in \R $, $ \norm{x} $, $ \abs{x} $, $ u\upind{ref}, \eye, \eyed{n\ind{x}}, \zeros $
\item For nice Git diffs, it is ideal to put one sentence in one line of \LaTeX~code.
\item Formulate your optimization problem as clear as possible.
We recommend using \texttt{optidef} to define them in \LaTeX, for example:
\begin{mini!}
	{x(\cdot),u(\cdot)}{\int_{0}^{T} \! L(t,x(t),u(t)) \, \mathrm{d}t + M(x(T)) \label{eq:nmpc_ocp_obj} }
	{\label{eq:nmpc_ocp}}{}
	\addConstraint{0}{= x(0)-\bar{x}_0  }
	\addConstraint{ 0 }{= f(t,x(t),\dot{x}(t), u(t)) }
	%	\addConstraint{ 0 }{= g(t,x(t), u(t)) \label{eq:ocp_eq}}
	\addConstraint{ 0 }{\geq h(t,x(t),u(t)) \quad \text{for all } t \in [0,T]. \label{eq:ocp_ineq}}
\end{mini!}
\end{itemize}
